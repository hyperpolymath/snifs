= snifs — Quick Start for Maintainers
:toc:
:toclevels: 2

== Overview

This guide covers building, verifying, packaging, and releasing snifs. snifs is a
research + reference repository (paper + BEAM/WASM implementation + proofs), so
"release" means: green proofs and tests, a rebuilt paper, a signed container bundle,
and a tagged version on Zenodo — not shipping an installable end-user application.

== Runtime Dependencies

* Elixir 1.15+ on OTP 25+ (host); `wasmex` → `wasmtime` (sandbox).
* Zig 0.15+ and a Rust→wasm32 toolchain (to rebuild guests).
* Idris2 / Lean4 / Agda (to re-check proofs); `just`; Guix or Nix.

== Build from Source

[source,bash]
----
git clone https://github.com/hyperpolymath/snifs.git
cd snifs
just build-wasm          # guests → priv/*.wasm (ReleaseSafe)
just build-release       # release build
----

== Pre-Release Gate

[source,bash]
----
just quality             # fmt-check + lint + test
just proof-check-all     # all proofs check
just abi-conformance     # model ↔ binary ABI drift gate
just assail              # panic-attacker security scan
just deps-audit          # dependency vulnerability audit
just sbom                # generate the software bill of materials
----

== The Paper

[source,bash]
----
just paper               # build docs/whitepapers/academic/snif.pdf
----

== Container (cerro-torre pipeline)

[source,bash]
----
just container-build     # build the image
just container-verify    # verify the compose configuration
just container-sign      # build + pack + sign + verify the bundle
just container-push      # push the signed bundle to the registry
----

The container build keys off the repo-root `Containerfile`; the
`dogfood-gate` workflow checks for it there — do not relocate it.

== Cutting a Release

[source,bash]
----
just changelog-preview   # preview unreleased changelog entries (no write)
just changelog           # generate CHANGELOG.md with git-cliff
just release-tag <version>
----

Tags and the Zenodo DOI are how downstream consumers cite a fixed version (see the
`Citation` block in `README.adoc`).
